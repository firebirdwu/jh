稽核系统

背景

中国大地财险稽核系统根据监管要求，与2008年建设，2012年升级。 2018年保监会和银监会合并成立银保监会后，于2019年引入保险现场检查系统（简称“EAST”系统）对保险公司报送的稽核数据进行检查 。 随监管要求日益严格以及EAST系统的引入，稽核当前系统已难以满足监管报送的时效性以及准确性要求，故重新设计稽核系统进行报送

目标

根据中国银保监会办要求，各保险公司按照相关接口文档，完成平台接口程序的开发工作。为了能够方便、快捷、安全、高效地实施系统对接，尽最大可能降低系统实施风险，结合大地财险业务情况和系统现状，提供成熟的IT建设方案，完成稽核系统建设目标。

业务术语

  • 【平台】: 指中国银保监会稽核系统登记平台，以下如不做特殊说明，简称为平台 。
  • 【稽核系统 】: 部署在保险公司业务系统与平台之间 的各保险公司自身的数据系统。
  • 【业务系统】: 指保险公司内部的需要跟平台实现对接的系统 。

  • 说明

    稽核系统主要包括数据提取和打包上传两个子系统，数据提取子系统主要负责按平台要求的提数口径和业务模型提取转换相关业务数据到目标表中；打包上传子系统主要负责把目标表中数据导出成平台要求的文件格式，完成上传、通知各分机构进行下载，上传当地平台 。

    数据提取子系统包括数据的预处理、业务数据提取、数据校验、数据修复等模块，将采用过程化SQL语言实现功能逻辑，通过Informatic配置调度完成数据提取 。

    打包上传子系统包括批次任务初始化、数据包文件生成、数据包压缩加密、上传、上传成功通知等模块，此系统将采用shell脚本的形式进行作业，通过Informatic远程执行shell脚本完成数据打包、加密、上传操作 。

    数据提取子系统PLPGSQL程序部署,且将部署在MPP数据平台（GREENPLUM）。打包上传子系统当前会使用shell脚本进行rar打包加密，生成随机密码，并上传SFTP服务器，供地方分机构进行提取 。

  • 系统流程

    保单登记系统主体业务流程将由Informatic调度触发，先依次执行数据提取子系统各模块，数据提取完成后，Informatic再通过执行远程shell，启动打包上传子系统，完成数据上传 。

    数据提取子系统模块执行:

    1. 预处理模块：主要针对复杂统计指标的表进行合并预处理，将指定指标预先进行处理，以供后续提数使用

      1. 业务提数模块：主要根据平台业务模型，结合大地财险业务情况和系统现状生成保送数据目标表 。
      2. 校验模块：根据平台提供校验规则，对数据有效性、完整性、合法性、表间的勾稽关系进行校验 。
      3. 数据修复：对异常数据信息进行修复，根据校验结果，定位异常数据表以及校验不通过指标，进行问题分析，并修复数据再次进行校验，直至校验合格
      4. 数据卸载：根据各分机构当地平台要求，将对应目标表数据按照平台要求生成对应上报文件。

    打包上传子系统模块执行:

    1. 格式确认：因稽核系统数据需提供给各分机构，并上传当地平台，因各地方平台对文件的格式、命名各不相同、故打包数据前需要对数据文件的名称、日志文件的名称进行检查 。
    2. 数据打包：合格后的数据需要进行压缩打包，以便数据的传输 。
    3. 数据加密：目前会将压缩文件压缩时使用随机密码（后续会使用indigo进行报文加密方式）生成加密后的压缩文件 。
    4. 文件上传：压缩后的文件会上传对应sftp，以供各分机构进行下载 。
    5. 消息通知：上传完成的机构数据，会以邮件的形势通知运维人员以及各对应分机构负责人
